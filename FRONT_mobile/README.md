# FRONT_mobile

Frontend-концепт для `Salon Svyazi` по ТЗ из `тз_кодекс.docx`.

## Что внутри

- `React + TypeScript + Vite`
- маршруты под роли: гость, клиент, сотрудник, администратор
- редакционный визуальный стиль по референсу: тёмные панели, молочный фон, крупная антиква
- моковый auth-state без подключения к реальному REST API
- подготовленные типы и `apiMap` для будущей интеграции

## Что уже сделано

- витрина устройств, тарифов и акций
- макет личного кабинета клиента
- макет панели сотрудника
- макет панели администратора
- демо-модалка входа и ручное переключение ролей

## Что пока не сделано

- нет реальной интеграции с `REST_mobile`
- нет настоящей авторизации
- нет вызовов PostgreSQL функций через backend
- не заведены реальные CRUD-формы

## Запуск

```bash
npm install
npm run dev
```

## Будущая интеграция

Текущая заготовка ориентируется на следующие маршруты API:

- `POST /auth/login`
- `GET /devices`
- `GET /tariffs`
- `GET /promotions`
- `GET /profile`
- `GET /repairs`
- `GET /contracts`
- `GET /bonus`
- `POST /sales`
- `POST /repairs`
- `PUT /repairs/{id}/close`
- `POST /contracts`
- `GET /employees`

Сейчас это только UI-слой. Бэкенд и фронтенд намеренно не связаны.
